Control Protocol/Internet Protocol

When systems communicate with each other, there are certain standards or protocols that allow them to transmit and receive data in an orderly manner. One of the most commonly used protocol suites around the world is TCP / IP (Transmission Control Protocol / Internet Protocol). (However, much of Europe uses the X.25 protocol.) Some common functions for using TCP / IP are e-mail, system-to-system file transfer, and remote login. The mail user command, Message Handling (MH) user commands, and the sendmail server command can use TCP / IP to send and receive mail between systems, and Basic Network Utilities (BNUs) can use TCP / IP to send and receive files and commands between systems.

TCP/IP

TCP / IP enables communications between multiple systems (called hosts) connected on a network. In turn, each network can be connected to another to communicate with the main systems of that network. Although there are many types of network technologies, many of which use packet-switched and streaming transport, TCP / IP offers one important advantage: hardware independence. Since Internet protocols define the unit of transmission and specify how to send it, TCP / IP can hide the details of network hardware, allowing many types of network technologies to connect and exchange information. Internet addresses make it possible for all machines on the network to communicate with each other. TCP / IP also provides standards for many of the communication services that users need.

Components

Sometimes it is called the TCP / IP protocol suite, in reference to the two most important protocols that make it up, which were among the first to be defined, and which are the two most used of the family:
TCP: transmission control protocol.
IP: internet protocol.
There are so many protocols in this set that there are more than a hundred different ones, among them are:

  1. ARP: address resolution protocol, to find the physical address (MAC) corresponding to a certain IP.
  2. FTP: file transfer protocol, popular in file transfer.
  3. HTTP - Hypertext transfer protocol, which is popular because it is used to access web pages.
  4. POP: post office protocol, for email.
  5. SMTP: Simple Mail Transfer Protocol for e-mail.
  6. Telnet (Telecommunication Network), to access remote computers.
  7. IDENTD: (Identification Daemon)
  8. IRC: (Internet Relay Chat)
  9. UDP: (User Datagram Protocol) User Datagram Protocol
  10. ICMP: (Internet Control Message Protocol)
  11. TFTP (Trivial FTP) Trivial FTP
  12. DHCP: (Dynamic Host Configuration Protocol) Dynamic Host Configuration Protocol
  13. NTP: (Network Time Protocol) Network Time Protocol
  14. DNS: (Domain Name Service) Domain Name Service
  15. SNMP: (Simple Network Management Protocol)
  16. RIP: (Routing Information Protocol)
  17. PPP: (Point to Point Protocol)

History

It was first developed and demonstrated in 1972 by the United States Department of Defense, running on the ARPANET, a wide-area network of that department.The Internet protocol family can be described by analogy with the OSI (Open System Interconnection) model, which describes the levels or layers of the protocol stack, although in practice it does not exactly correspond to the Internet model. In a protocol stack, each layer solves a series of tasks related to data transmission, and provides a well-defined service at the highest levels. The higher levels are closest to the user and deal with more abstract data, leaving the lower levels the task of translating the data so that it is physically manipulable. The Internet model was designed as the solution to a practical engineering problem. The OSI model, on the other hand, was proposed as a theoretical approach and also as a first phase in the evolution of computer networks. Therefore, the OSI model is easier to understand, but the TCP / IP model is the one that is actually used. It helps to understand OSI before you know TCP / IP, as the same principles apply, but are easier to understand in OSI.

Advantages and disadvantages

The TCP / IP suite is designed to route and has a very high degree of reliability, suitable for networks of any size. It is used globally on all types of networks, from home to business networks and on the Internet. It is compatible with standard tools for analyzing network performance.One drawback of TCP / IP is that it is more difficult to configure and maintain than NetBEUI or IPX / SPX; furthermore, it is somewhat slower on networks with low medium traffic volume. However, it can be faster on high-traffic networks where a large number of frames have to be routed.
